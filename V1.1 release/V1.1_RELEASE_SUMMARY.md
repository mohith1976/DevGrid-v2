## What's New in V1.1

### 🎯 Major Features

#### 1. Reset Statistics
Users can now clear all cached data while keeping their GitHub connection intact.

**Benefits**:
- Start fresh without reconnecting
- Clear corrupted cache data
- Test sync functionality
- Rebuild statistics from scratch

**Implementation**:
- New "Reset Statistics" button in popup
- Confirmation dialog with clear explanation
- Clears all caches except GitHub config
- Updates GitHub README to show zero stats
- Success feedback after completion

---

#### 2. Repository Metadata Auto-Setup
Repositories now get professional metadata automatically after first sync.

**What Gets Set**:
- **Description**: "LeetCode solutions synchronized automatically using DevGrid."
- **Homepage**: `https://github.com/mohith1976/DevGrid-v2`
- **Topics**: `leetcode`, `algorithms`, `dsa`, `chrome-extension`, `typescript`, `devgrid`

**Smart Behavior**:
- Only updates if description is empty
- Never overwrites user customizations
- Runs once per repository
- Non-blocking (sync succeeds even if metadata update fails)

**Benefits**:
- Professional repository appearance
- Better discoverability on GitHub
- Attribution to DevGrid project
- Consistent branding

---

#### 3. Rich Problem READMEs
Problem READMEs now include complete problem details extracted from LeetCode.

**What's Included**:
- Full problem description
- All examples with input/output
- All constraints
- Topic tags (when available)
- Runtime and memory percentiles
- Solution code with syntax highlighting

**Benefits**:
- Self-contained documentation
- No need to revisit LeetCode to understand problem
- Better for portfolio/interview prep
- Complete reference material

---

#### 4. Synced Problem Indicators
Visual indicators on LeetCode show which problems are already backed up.

**Features**:
- "✓ Synced to GitHub" badge on problem pages
- GitHub icon for visual recognition
- "View" link to GitHub folder
- Survives SPA navigation
- Uses local cache (no API calls)

**Benefits**:
- Know what's backed up at a glance
- Quick access to GitHub from LeetCode
- No duplicate work
- Better workflow awareness

---

#### 5. Repository About Section
Extension popup now shows repository description.

**Features**:
- Displays repository description
- Fallback text for empty descriptions
- Fetched during GitHub connection
- Updates after metadata setup

**Benefits**:
- More informative popup
- Confirms repository connection
- Professional appearance

---

### 🎨 UI/UX Improvements

#### Popup Redesign
- Removed purple gradients and emojis
- Clean black/white/gray color system (#1a1a1a, #737373, #a3a3a3)
- Reduced font size to 13px
- Improved spacing and hierarchy

#### Sync Status Overlay
- Three states: uploading (blue), success (green), error (red)

- Auto-dismiss for success (3 seconds)

#### Repository README
- Removed emoji spam and decorative ASCII
- Clean sections: Statistics, Languages, Top Topics, Recent Activity
- Visual language bars
- Professional formatting

#### Problem README
- Simplified template
- Removed excessive badges
- Clean structure
- Includes performance percentiles

---

### 🔧 Technical Improvements

#### Commit Message Format
**Old**: `Add Two Sum (Python)`  
**New**: `Time: 45 ms (95.23%), Space: 14.2 MB (87.45%) - DevGrid`

**Benefits**:
- Performance metrics at a glance
- Better commit history
- Easier to track improvements

#### Topic Extraction
Multi-tier fallback system:
1. GraphQL API topics (primary)
2. DOM extraction from problem page (fallback)
3. Hide Topics section if unavailable (never shows "Topics: None")

**Benefits**:
- More problems have topics
- Better categorization
- Cleaner READMEs

#### Last Sync Display
Fixed "NaN days ago" error:
- Changed from `new Date().getTime()` to `Date.now()`
- Improved time format: "5m ago", "2h ago", "Yesterday", "3d ago"
- Added "No successful sync yet" for empty state

---

### 🐛 Bug Fixes

#### Invalid Time Value Error
Fixed "Invalid time value" error in repository README generation:
- Added timestamp validation
- Migration logic for old cached data
- Default to current time if invalid

#### Build Configuration
Fixed ES module issues:
- Separate Vite configurations for each entry point
- Content script and popup bundled as IIFE
- Background script as ES module
- Proper file copying automation

### Unchanged from V1.0

2. Manual token management (no OAuth)
3. Single repository support
4. No conflict resolution
5. No rate limit handling
6. Network dependency (no offline queue)

## Upgrade Instructions

### For Existing Users

**No breaking changes**. Simply reload the extension:

1. Navigate to `chrome://extensions/`
2. Find DevGrid
3. Click the reload icon (circular arrow)
4. All existing data preserved

**To Get New Features**:
- Reset statistics button available immediately
- Repository metadata will be set on next sync
- Rich problem details will appear on next submission
- Synced indicators will appear on problem pages

**Optional**:
- Click "Reset Statistics" to trigger repository metadata update
- Resubmit old problems to get rich READMEs

## Security Review

### ✅ Security Measures
- Token stored in chrome.storage.local (isolated from web pages)
- Content script runs in isolated world
- Minimal permissions requested
- No external dependencies with security issues
- No eval() or unsafe code execution

### ⚠️ Security Considerations
- Token stored in plaintext (acceptable for V1.1, OAuth planned for V2.0)
- No token expiration handling
- No rate limit protection
